## 4. 종합 분석

### 4.1 일반적인 정책 패턴

#### A. 해커톤 (단기 이벤트)
- **IP 소유권**: 참가자 완전 소유
- **조직 권리**: 마케팅/프로모션 라이선스만 보유
- **상업화**: 자유 (제약 없음)
- **팀 분쟁**: 개인 간 해결 (조직 개입 최소)

#### B. 인큐베이터/액셀러레이터
- **IP 소유권**: 창업자 소유 (법인 구조)
- **조직 권리**: 지분 투자자로서 권리 (5-10%)
- **상업화**: 완전 자유 (투자 목적)
- **팀 분쟁**: 주주 계약으로 해결

#### C. 교육 기관
- **IP 소유권**: 학생 소유 (명시 또는 묵시적)
- **조직 권리**: 거의 없음
- **상업화**: 자유
- **팀 분쟁**: 개인 간 해결

---

### 4.2 "팀 프로젝트를 개인이 상업화하고 싶을 때" 처리 방법

#### 패턴 1: 사전 합의 (가장 일반적)
```
1. 프로젝트 시작 전 팀원 간 IP 배분 합의
2. 문서화 (간단한 합의서)
3. 상업화 시 합의 내용 준수
```

**적용 사례**: 대부분의 해커톤, 교육 기관

**장점**
- 사전 명확화로 분쟁 예방
- 유연한 합의 가능

**단점**
- 참가자들이 법률 지식 부족 시 문제
- 강제력 약함

---

#### 패턴 2: 법인 구조 활용
```
1. 프로젝트 시작 시 법인 설립
2. 지분 배분 명확화
3. 주주 계약으로 권리/의무 정의
4. 상업화 = 회사 운영
```

**적용 사례**: YC, TechStars 등 인큐베이터

**장점**
- 법적 명확성
- 강제력 있음
- 투자 유치 용이

**단점**
- 초기 비용 및 복잡성
- 스터디/교육 단계에는 과도함

---

#### 패턴 3: 조직 중재
```
1. 분쟁 발생 시 조직이 중재
2. 합의 도출 지원
3. 합의 실패 시 법적 절차 안내
```

**적용 사례**: 일부 인큐베이터, 교육 기관

**장점**
- 제3자 관점에서 공정한 중재
- 관계 유지 가능

**단점**
- 조직의 책임 및 부담 증가
- 법적 효력 제한적

---

#### 패턴 4: 기여도 기반 배분
```
1. 각자의 기여도 기록 (코드 커밋, 문서 등)
2. 상업화 시 기여도 비율로 수익 배분
3. GitHub 기여 이력 등 객관적 지표 활용
```

**적용 사례**: 오픈소스 프로젝트 참고

**장점**
- 객관적 지표 기반
- 공정성 확보

**단점**
- 기여도 측정 어려움 (아이디어, 디자인 등)
- 정량화 곤란한 요소 존재

## 5. 테코테코 적용 권장사항

### 5.1 기본 IP 정책 (권장안)

```markdown
## 테코테코 프로젝트 IP 정책 (안)

### 1. 기본 원칙
- 모든 프로젝트의 IP는 **참여자(팀원)가 공동으로 소유**합니다
- 테코테코는 프로젝트에 대한 **어떠한 소유권도 주장하지 않습니다**
- 테코테코는 마케팅 목적으로 프로젝트를 소개할 수 있는 **비독점적 라이선스**만 보유합니다

### 2. 상업화 규칙
#### A. 팀 전원 합의 상업화
- 팀원 전원이 합의한 경우, 자유롭게 상업화 가능
- 테코테코에 별도 통지 의무 없음
- 권장: 사전에 팀원 간 지분/역할 합의

#### B. 개인 상업화 (팀원 일부만 참여)
**필수 절차:**
1. 상업화를 원하는 팀원은 나머지 팀원에게 사전 통지
2. 30일 내 나머지 팀원의 서면 동의 획득
3. 동의 시: 자유롭게 상업화 진행
4. 거부 시: 해당 프로젝트 상업화 불가
5. 단, 프로젝트에서 본인이 작성한 코드를 새로운 프로젝트에 재사용 가능

**동의 서식 (예시):**
```
프로젝트명: [프로젝트명]
상업화 주체: [이름]
나머지 팀원: [이름1, 이름2...]

본인은 위 프로젝트의 상업화에 대해 동의합니다.
(또는) 본인은 위 프로젝트의 상업화에 대해 동의하지 않습니다.

서명: ___________  날짜: ___________
```

### 3. 테코테코의 권리
#### 허용되는 사용:
- 블로그/SNS에서 프로젝트 소개
- 데모데이/발표회에서 프로젝트 시연
- 홍보 자료에 프로젝트 포함

#### 제한사항:
- 상업적 판매 금지
- 제3자에게 라이선스 제공 금지
- 프로젝트 파생물 생성 금지

### 4. 팀원 간 분쟁 해결
#### 1단계: 자율 협의
- 팀원 간 직접 대화로 해결 시도
- 기간: 분쟁 발생 후 14일

#### 2단계: 테코테코 중재 (선택사항)
- 테코테코 운영진이 중재 역할 제공
- 법적 효력은 없으나 합의 도출 지원
- 신청: dispute@tecoteco.com

#### 3단계: 법적 절차
- 합의 실패 시 법률 전문가 상담 권장
- 테코테코는 법적 책임 없음

### 5. 권장 사항
#### 프로젝트 시작 시:
1. **팀원 간 IP 합의서 작성** (템플릿 제공)
2. GitHub Organization 생성 및 팀원 추가
3. 기여 이력을 명확히 기록 (커밋 메시지, 문서)

#### 템플릿 링크:
- 팀 IP 합의서 템플릿
- 상업화 동의서 템플릿
```

### 5.2 단계별 적용 방안

#### Phase 1: 즉시 적용 (필수)
1. **명확한 IP 정책 문서화**
   - 위 권장안 기반으로 테코테코 공식 정책 작성
   - 웹사이트에 게시

2. **참여 동의 절차**
   - 프로젝트 시작 시 정책 동의 받기
   - 간단한 체크박스 형태로 구현

## 7. FAQ (예상 질문)

### Q1. 프로젝트를 혼자 진행했어요. 그래도 합의서가 필요한가요?
**A:** 혼자 진행한 프로젝트는 본인이 100% 소유하며, 별도 합의서가 필요 없습니다.

### Q2. 팀원 중 한 명이 연락이 안 돼요. 어떻게 하나요?
**A:** 
1. 이메일, 메시지 등 여러 채널로 연락 시도
2. 30일 대기 후에도 연락 없으면 테코테코에 중재 요청
3. 최종적으로는 법률 자문 권장

### Q3. 기존 오픈소스 코드를 사용했어요. 상업화 가능한가요?
**A:** 사용한 오픈소스의 라이선스에 따라 다릅니다:
- MIT, Apache: 대부분 상업화 가능
- GPL: 파생물도 오픈소스 공개 필요
- 라이선스 확인 후 판단하세요

### Q4. 테코테코가 우리 프로젝트를 홍보에 사용하는 게 싫어요.
**A:** 개별적으로 요청하시면 홍보 자료에서 제외해드립니다. (privacy@tecoteco.com)

### Q5. 프로젝트를 회사에 제출해도 되나요?
**A:** 본인이 작성한 부분은 자유롭게 사용 가능합니다. 단, 팀원이 작성한 부분은 동의가 필요합니다.

### Q6. 나중에 더 많은 팀원이 참여하면 어떻게 하나요?
**A:** 새로운 팀원 합류 시마다 기존 합의서를 업데이트하고 모든 팀원이 재서명해야 합니다.

### Q7. 팀원과 의견이 안 맞아요. 테코테코가 결정해주나요?
**A:** 테코테코는 중재 역할만 제공하며, 최종 결정은 팀원들이 합의하셔야 합니다.

### Q8. 상업화해서 돈을 많이 벌면 테코테코에 수수료를 내야 하나요?
**A:** 아니요. 테코테코는 어떠한 수수료도 받지 않습니다.
