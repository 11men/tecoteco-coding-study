# PRD: 섀도우 포트폴리오 — "마이너스의 손"

**버전:** 2.0
**작성일:** 2026-02-10
**상태:** Draft

---

## 1. 배경 및 문제 정의

### 배경

개인 투자자의 가장 큰 손실 원인은 **FOMO(Fear of Missing Out)** 에 의한 충동 매수다. 기존 모의투자 게임은 "얼마나 많이 버냐"를 경쟁하지만, 현실에서 개인 투자자 대다수는 고점 매수 후 손실을 반복한다.

### 역발상

> "FOMO로 샀다면 어땠을까?"를 **실제 주가로 추적**하면, 대부분의 충동 매수는 손실로 끝난다.

이 사실을 게임으로 만들면:

```
일반 모의투자:      많이 벌수록 → 1위
섀도우 포트폴리오:  많이 잃을수록 → 1위 (= 현실에서 FOMO를 많이 방어한 것)
```

### 해결하려는 문제

| 문제 | 기존 해결책 |
|------|------------|
| FOMO 충동 매수 반복 | 없음. 기존 앱은 매수 촉진 구조 |
| "참았더라면"의 결과 추적 불가 | 없음 |
| 참는 행동에 대한 보상/경쟁 구조 없음 | 없음 |

---

## 2. 제품 개요

**섀도우 포트폴리오**는 사용자가 FOMO를 느꼈지만 실제로는 매수하지 않은 종목을 **가상으로 매수 처리**하고, 실제 주가 API로 연동하여 그 손익을 추적하는 역발상 모의투자 게임이다.

**가상 포지션은 무기한 보유**되며, 가상 잔고가 가장 적은 사람(= 가장 많이 잃은 사람)이 1위다.

## 3. 목표 및 KPI

### 비즈니스 목표
- 광고 시청 기반 수익 창출 (가상 잔고 재충전)
- 소셜 바이럴을 통한 유기적 성장 (유료 마케팅 비용 최소화)

### KPI

| 지표 | 목표 (출시 후 6개월) |
|------|---------------------|
| MAU | 50,000 |
| 1인당 일평균 광고 시청 횟수 | 2회 이상 |
| D7 리텐션 | 40% 이상 |
| D30 리텐션 | 20% 이상 |
| SNS 결과 카드 공유율 (DAU 대비) | 5% 이상 |

## 4. 타겟 유저

### Persona A — "충동 매수 반성러" (주 타겟)

- **연령:** 20~35세
- **특징:** 코인·주식 소액 투자 경험 있음. 충동 매수 후 손실 반복. 투자 커뮤니티 활동(클리앙, 주식 갤러리, 코인 오픈채팅)
- **니즈:** "내가 참았던 게 맞는 선택이었는지 확인하고 싶다"
- **게임 동기:** "나 오늘 모의투자로 150만원 잃었어 ㅋㅋ" → SNS 공유 본능

### Persona B — "자기 통제 챌린저"

- **연령:** 25~40세, 직장인
- **특징:** 장기 투자 지향. 단타 유혹을 스스로 통제하려 함
- **니즈:** "참는 기록을 데이터로 보고 싶다"
- **게임 동기:** 랭킹 경쟁보다 자기 기록 추적

## 5. 핵심 게임 메커니즘

### 5.1 가상 잔고 시스템

```
가입 시   → 가상 잔고 1,000,000원 지급 (1회)
광고 시청 → 가상 잔고 1,000,000원 재충전 (횟수 제한 없음)
```

- 가상 잔고는 현금이 아닌 앱 내 게임 재화
- 실제 돈과 교환 불가 (사행성 규정 회피)

### 5.2 섀도우 기록 (가상 매수)

사용자가 FOMO를 느낀 종목을 기록하면 **즉시 가상 매수 처리**된다.

**입력값:**
- 종목 (국내주식 / 해외주식 / 코인 검색)
- 가상 투자 금액 (잔고 내에서 자유 설정)

**처리:**
```
기록 시점의 실제 종가를 매수가로 스냅샷 저장
→ 해당 금액만큼 가상 잔고에서 즉시 차감
→ 이후 실제 주가 API로 연동하여 실시간 평가손익 반영
```

### 5.3 실제 주가 연동 및 평가손익

| 상황 | 가상 잔고 | 랭킹 |
|------|----------|------|
| 보유 종목 하락 | 감소 | 상승 (유리) |
| 보유 종목 상승 | 증가 | 하락 (불리) |

- **포지션 청산 없음** — 무기한 보유
- 잔고가 0원 이하로 내려갈 수 있음 (마이너스 표시)
- 마이너스 잔고 상태에서도 광고 시청으로 재충전 가능

### 5.4 랭킹 기준

```
랭킹 점수 = 누적 가상 손실액 (클수록 높은 순위)
          = 최초 충전 합계 - 현재 가상 잔고
```

동점 시: 기록 건수 많은 유저 우선

## 6. "마이너스의 손" 타이틀 시스템

마이다스의 손(황금 손)의 패러디. 건드리는 종목마다 폭락시키는 사람이 이 게임의 MVP.

### 레벨 구조

| 레벨 | 이름 | 조건 |
|------|------|------|
| Lv.1 | 펄럭귀 (Paper Ear) | 첫 기록 |
| Lv.5 | 하락 탐지기 | 누적 손실 50만원 |
| Lv.10 | 돌부처 (Stone Buddha) | 누적 손실 200만원 |
| Lv.30 | 공매도의 신 | 누적 손실 1,000만원 |
| Lv.99 | 마이너스의 손 | 누적 손실 1억원 |

### 배지

| 배지명 | 조건 |
|--------|------|
| 생존자 | 코스피 -3% 이상 폭락일 — 기록 추가 없이 버팀 |
| 폭락 예언자 | 기록 후 1주일 내 -20% 이상 하락 |
| 마이너스의 손 | 단일 종목 가상 손실 100만원 초과 |
| 챌린지 완주자 | 챌린지 기간 전체 기록 유지 |

## 7. JOMO Calculator

방어한 손실(가상 손실액)을 실물 가치로 환산.

```
가상 잔고 -500,000원 하락 시:
→ "치킨 25마리를 공짜로 먹은 것과 같습니다"
→ "스타벅스 아메리카노 166잔"
→ "제주도 편도 항공권 2장"
```

- 환산 기준 단가: 분기별 업데이트
- 공유 카드에 포함되는 핵심 바이럴 요소

## 8. 소셜 챌린지

### 개요

월 1~2회 운영되는 **무료 테마 챌린지**. 챌린지 기간 중 가장 많이 잃은 사람이 우승.

### 챌린지 유형 예시

| 챌린지명 | 조건 | 기간 |
|---------|------|------|
| 폭락장 생존자 챌린지 | 지수 급락 기간 중 누적 손실 최대화 | 이벤트성 |
| 이달의 마이너스의 손 | 월간 가상 손실 최대 | 월간 |
| 코인 FOMO 챌린지 | 코인 종목만 대상 | 2주 |

### 결과 공유 카드 (바이럴 핵심)

```
┌─────────────────────────────────┐
│  📉 이달의 마이너스의 손          │

│                                 │
│  랭킹: 전체 23위 / 4,821명       │
│  레벨: Lv.23 돌부처              │
```

## 9. 수익화 모델

### 광고 (핵심 수익원)

| 트리거 | 광고 유형 | 보상 |
|--------|----------|------|
| 가상 잔고 소진 | 보상형 전면 광고 | 가상 100만원 재충전 |
| 챌린지 참여 | 선택형 광고 시청 | 추가 기록 슬롯 |
| 결과 카드 공유 전 | 건너뛸 수 없는 5초 광고 | 공유 실행 |

### 광고 수익 추정 (보수적)

```
MAU 50,000명 × DAU 전환율 30% = 일 활성 15,000명
일 평균 광고 시청 2회 × eCPM 3,000원 / 1,000회
= 일 수익 약 90,000원 → 월 약 2,700,000원

MAU 10만 달성 시 월 약 5,400,000원 예상
```

## 10. 사용자 플로우

```
신규 유저 진입
  → 온보딩: "FOMO로 산 적 있나요?" (2단계)
  → 가상 잔고 1,000,000원 지급
  → 첫 섀도우 기록 작성 유도
  → 실시간 가상 손익 확인 (Aha Moment)
  → 잔고 소진 → 광고 시청 → 재충전
  → 챌린지 배너 노출 → 참여
  → 결과 카드 SNS 공유 → 신규 유저 유입
```

## 11. 기술 요구사항

### 주가 데이터 API

| 자산군 | API 후보 |
|--------|---------|
| 국내 주식 | KIS Developers (한국투자증권) |
| 해외 주식 | Alpha Vantage, Yahoo Finance |
| 코인 | CoinGecko (무료 티어) |

- 기록 시점 **종가 스냅샷 저장** 필수 (매수가 기준)
- 평가손익: 장중 지연 허용 (15분), 종가 기준 일 1회 갱신

### 가상 잔고

- 서버 사이드 관리 필수 (클라이언트 조작 방지)
- 마이너스 잔고 허용 (음수 표현 지원)

### 어뷰징 방지

- 동일 종목 동일일 중복 기록 차단
- 동일 디바이스 다계정 탐지
- 광고 시청 시 유효성 검증 (광고 SDK 콜백 기반)

## 12. 법적 / 규제 검토

### 사행성 문제 없음 (구조적 해소)

- 가상 잔고는 게임 내 재화 — 현금 전환 불가
- 상금 없음 — 배지·레벨 등 비금전 보상만 지급
- 실제 투자 조언 아님 — 면책 조항 명시 필수

### 투자 조언 오해 방지

```
앱 내 상시 표시:
"본 앱은 투자 조언을 제공하지 않습니다.
모든 기록은 사용자 개인의 가상 기록이며,
실제 투자 결과와 다를 수 있습니다."
```
