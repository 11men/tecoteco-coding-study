**서울/경기 버스 파업 알림 및 대체 경로 안내**

핵심 인사이트: *"파업 알림"은 뉴스가 해준다. 우리가 해야 할 건 **"그래서 나는 어떻게 출근해?"**에 답하는 것*

- **팀원**: 최병호, 홍영훈, 정석호

- **기술**: Next.js 16, React 19, TypeScript, v0
- **특징**: 동작하는 프로토타입 완성, 커뮤니티/게이미피케이션 포함

- **프로젝트**: [Matchmaking Server](./team2/)

- **기술**: Node.js, Redis, Lua Script (원자적 매칭)
- **특징**: 지역별 대기열, N명 충족 시 즉시 매칭, 동시성 안전 보장

### Team 3: 버스 파업 알림 앱 (풀스택 설계)

- **기술**: Spring Boot 3.4 (Java 21), React 18, PostgreSQL, Redis, AWS
- **특징**: 10개 기능 로드맵, 크롤링 시스템, 푸시 알림

## 🎤 팀별 발표 요약

### Team 1: 파업 출근 비서
**핵심 전략**
- "파업 알림"은 뉴스가 해준다 → 우리는 **"어떻게 출근할지"** 답한다
- 리텐션 문제 해결: 파업 시만 → **365일 사용**하는 출퇴근 플랫폼

**구현 결과**
- MVP: 경로 등록, 파업 알림, 대체 경로, 소요시간 비교
- 확장: 출근 브리핑, 실시간 교통 이슈, 커뮤니티, 택시/카풀, 포인트/배지

**피드백**
- ✅ 비즈니스 관점 접근 인상적 (수익 모델까지 고려)
- ✅ 상시 가치 제공으로 리텐션 문제 해결
- ⚠️ 프론트엔드 목업만 완성, 백엔드 연동 필요

### Team 2: 카풀 매칭 서버
**핵심 전략**
- 파업 시 즉시 매칭되는 카풀 인프라
- Redis + Lua Script로 **동시성 안전 보장**

**구현 결과**
- 지역별 대기열 관리 (FIFO)
- minN명 충족 시 즉시 매칭 생성
- Pub/Sub으로 실시간 알림
- 동시성 테스트 (100명 동시 요청) 통과

**피드백**
- ✅ Lua 스크립트로 원자적 처리 구현 (기술적 완성도 높음)
- ✅ 실제 동작하는 코드 + 테스트 완성
- ⚠️ UI 없음, 지도/위치 연동 부재

### Team 3: 버스 파업 알림 앱
**핵심 전략**
- 엔터프라이즈급 **완전한 시스템 설계**
- 30분 주기 크롤링으로 자동 파업 정보 수집

**구현 결과**
- 10개 기능 로드맵 (P0/P1/P2 우선순위)
- Spring Boot + React 프로젝트 구조
- 크롤링 시스템 설계 (TOPIS/GBIS)
- DB 스키마, API 명세, 인프라 설계 (AWS)

**피드백**
- ✅ 체계적 문서화 (20개 이상 마크다운)
- ✅ 프로덕션 레디 아키텍처
- ⚠️ 2시간 내 코드 완성도 부족, UI 미구현

## 💭 전체 회고

### 잘했던 점 (Keep)
- **다양한 접근**: 프로토타입(Team 1) vs 인프라(Team 2) vs 설계(Team 3)
- **AI 활용**: 문서 생성, 코드 스캐폴딩, UI 프로토타이핑
- **2시간 집중**: 짧은 시간 내 의미 있는 결과물

### 아쉬운 점 (Problem)
- **시간 부족**: 2시간으로는 완성도 있는 구현 어려움
- **협업 도구**: Claude 세션 공유로 병목 발생
- **통합 부족**: 각 팀이 독립적으로 진행, 시너지 부족

### 다음에 시도할 점 (Try)
- **역할별 분리**: 기획/프론트/백엔드 세션 분리
- **실제 연동**: 목업이 아닌 실제 동작하는 시스템
- **팀간 협업**: 서로의 결과물을 조합하는 방식 시도
