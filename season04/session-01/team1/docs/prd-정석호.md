# PRD: 서울/경기 버스 파업 알림 서비스 v2.0

## 1. 배경 및 문제 정의

서울/경기 지역의 버스 파업은 출퇴근·등교 등 일상 이동에 직접적인 영향을 미치지만,

- 파업 정보가
  - 서울시·경기도청 보도자료
  - 지자체/버스조합 공식 SNS
  - 문자 안내
  - 뉴스 기사
    등 **여러 채널에 분산**되어 있음
- "내가 이용하는 지역/버스가 영향 대상인지"를 **즉시 판단하기 어려움**
- 파업 발생 사실을 **늦게 인지**하거나, **불완전한 정보로 잘못 대응**하는 경우가 잦음

현재 파업 정보를 **통합·요약·개인화하여 전달하는 서비스는 부재**함.

---

## 2. 목표 (Goals)

### 2-1. 제품 비전

서울/경기 버스 파업 발생 시,
**사용자가 아무것도 검색하지 않아도
→ 푸시 알림과 요약 페이지를 통해
→ "어디에서 어떤 버스가 영향을 받는지"를 즉시 알 수 있도록 한다.**

---

### 2-2. MVP 핵심 목표

1. 서울/경기 지역 버스 파업 발생(확정 기준) 시 **푸시 알림 제공**
2. 푸시 알림 클릭 시 **전용 페이지에서 파업 범위 명확히 안내**

### 2-3. 후속 기능 목표

3. 파업으로 출퇴근이 어려운 경우 **대체 이동 수단 정보 제공**

---

## 3. 타겟 사용자

### 주요 타겟

- 서울/경기 지역 대중교통 이용자
- 출퇴근 시간대 버스 의존도가 높은 직장인·학생
- 특정 이동 루트를 반복적으로 이용하는 사용자

### 사용자 세그먼트별 니즈

- **직장인**: 출근 전 사전 대응 필요 (전날 밤 ~ 당일 오전 6시)
- **학생**: 등교 시간 유연성 낮음 (빠른 알림 필수)
- **시니어**: 대체 수단 활용 어려움 (명확한 안내 필요)

## 4. 파업 정보 수집 방식

### 4-1. 정보 출처

버스 파업 정보는 아래 경로를 통해 확인한다.

**1차 출처 (최우선)**

- 서울시 / 경기도청
  - 공식 홈페이지 보도자료
  - 문자 안내
  - 공식 SNS (서울시 교통정보 공식 계정 등)
- 지자체 및 버스조합 공식 발표

**2차 출처 (교차 검증용)**

- 주요 언론 보도 (연합뉴스, KBS, MBC, SBS 등)

**출처 간 정보 상충 시 우선순위**

1. 서울시/경기도청 공식 발표
2. 버스조합 공식 발표
3. 언론 보도 (복수 언론사 교차 확인)

---

### 4-2. 수집 프로세스

**모니터링 담당**

- 운영팀이 수동 모니터링
- (선택) 향후 자동 크롤링 시스템 구축 검토

**모니터링 주기**

- 평시: 1일 2회 (오전 9시, 오후 6시)
- 파업 예고 시: 1시간 간격
- 파업 당일: 30분 간격

**정보 지연 목표**

- 공식 발표 후 1시간 이내 시스템 반영
- 파업 확정 후 15분 이내 푸시 알림 발송

**야간/주말 대응**

- 파업 예고가 없는 경우: 일 1회 모니터링 (오전 9시)
- 파업 예고가 있는 경우: 담당자 비상 연락망 가동

---

### 4-3. 정보 신뢰 기준

- 공식 기관 또는 언론을 **1차 출처**로 사용
- 추측·가능성 단계 정보는 **상태값으로 명확히 구분**
- 확정되지 않은 정보는 푸시 알림 발송 대상에서 제외
- SNS 루머 및 미확인 정보는 절대 반영하지 않음

---

## 5. 파업 상태 관리

### 5-1. 상태 정의 (우선순위: High)

파업 정보는 반드시 상태값으로 관리한다.

| 상태                   | 설명                                     | 푸시 알림 발송                          |
| ---------------------- | ---------------------------------------- | --------------------------------------- |
| **파업 예고**          | 파업 가능성이 공식 발표됨 (협상 진행 중) | ❌ 미발송                               |
| **파업 확정**          | 파업 실시가 확정됨 (협상 결렬)           | ✅ 발송                                 |
| **부분 파업**          | 일부 노선/시간대만 운행 중단             | ✅ 발송                                 |
| **지역 한정 파업**     | 특정 구/시에만 영향                      | ✅ 발송 (조건부)                        |
| **파업 종료 (정상화)** | 파업이 종료되고 정상 운행 재개           | ✅ 발송                                 |
| **파업 취소**          | 예고 후 파업이 취소됨                    | ✅ 발송 (예고 알림을 받은 사용자에게만) |

---

### 5-2. 상태 전환 규칙

```
[파업 예고] → [파업 확정] → [파업 종료]
            ↘ [파업 취소]

[파업 예고] → [부분 파업] → [파업 종료]
```

**주의사항**

- 상태 전환 시 이전 알림 수신자에게 업데이트 알림 발송
- 연속 파업 시 중복 알림 방지 로직 필요 (24시간 내 동일 상태 중복 알림 차단)

## 6. 푸시 알림 발송 로직

### 6-1. 발송 조건 (우선순위: High)

| 파업 범위           | 발송 대상          | 비고                  |
| ------------------- | ------------------ | --------------------- |
| **서울 전역 파업**  | 전체 서울 사용자   | 무조건 발송           |
| **경기 전역 파업**  | 전체 경기 사용자   | 무조건 발송           |
| **특정 구/시 파업** | 해당 지역 사용자만 | 위치 정보 기반 필터링 |
| **특정 노선 파업**  | 해당 노선 이용자만 | 사전 등록된 루트 기반 |

**위치 정보가 없는 사용자 처리**

- MVP: 서울/경기 전역 파업만 알림 발송
- 후속: 온보딩 시 "주요 이용 지역" 선택 기능 추가

---

### 6-2. 알림 문구 예시

**파업 확정 시**

```
[버스 파업 알림] 내일(1/28) 서울 시내버스 파업
→ 영향 지역 및 대체 교통 확인하기
```

**부분 파업 시**

```
[버스 파업 알림] 내일(1/28) 오전 6~9시 일부 노선 운행 중단
→ 내 노선 확인하기
```

**파업 종료 시**

```
[버스 정상 운행] 오늘(1/28) 오후 2시부터 정상 운행 재개
```

**파업 취소 시**

```
[파업 취소] 예정된 버스 파업이 취소되었습니다
```

---

### 6-3. 발송 시간 정책

**기본 원칙**

- 평일 출근 영향: 전날 오후 6시 ~ 밤 10시 발송
- 주말/공휴일 영향: 당일 오전 6시 발송
- 긴급 상황: 즉시 발송

**Do Not Disturb 고려**

- 밤 10시 ~ 오전 6시: 발송 제한 (단, 긴급 상황 제외)
- 사용자가 "긴급 알림 허용" 설정 시 예외

## 7. 파업 상세 페이지

### 7-1. 필수 정보 (우선순위: High)

푸시 알림 클릭 시 진입하는 전용 페이지에 다음 정보를 제공한다.

#### 1) 파업 상태 표시

```
[빨간 배지] 파업 진행 중
[주황 배지] 부분 파업
[초록 배지] 정상 운행
```

#### 2) 파업 발생 지역

- 서울 전역 / 특정 구 (예: 강남구, 노원구)
- 경기 전역 / 특정 시·군 (예: 고양시, 성남시)
- 지도 기반 시각화 (선택 기능)

#### 3) 영향받는 버스 범위

- 시내버스 / 광역버스 / 마을버스
- (가능한 경우) 노선 그룹 또는 번호대
  - 예: "간선버스(파란색) 전체", "지선버스(초록색) 일부"
  - 예: "101~199번 노선"

#### 4) 대체 운행 정보

- 대체 버스 운행 여부
- 감차 비율 (예: "평소 대비 50% 운행")
- 운행 시간대 제한 (예: "출퇴근 시간대만 운행")

#### 5) 메타 정보

- 정보 기준 시각 (예: "1월 27일 오후 3시 기준")
- 정보 출처 (예: "서울시 교통정책과 공식 발표")
- 마지막 업데이트 시각

## 8. 대체 이동 수단 안내 (후속 기능)

### 8-1. 전제 조건 (우선순위: Medium)

**필수 데이터**

- 사용자가 사전에 "출퇴근 루트"를 등록해야 함
- 또는 위치 기반 추론 (개인정보 수집 동의 필요)

**온보딩 플로우 추가 필요**

1. 앱 설치 후 "주요 이동 루트 등록하기" 유도
2. 출발지/도착지 입력 (주소 또는 지하철역)
3. 평소 이용 버스 노선 선택 (선택 사항)

**데이터 없는 사용자 대응**

- MVP: 일반적인 대체 수단 안내 제공
  - 예: "지하철 2호선, 9호선 이용 가능"
- 후속: "루트 등록하고 맞춤 대체 수단 받기" CTA 표시

---

### 8-2. 제공 정보 (우선순위: Medium)

#### 제공 내용

- 인근 지하철 노선 및 역 정보
- 파업 영향 없는 다른 버스 노선
- 환승이 필요한 경우 간단한 이동 요약
- 도보 가능 거리인 경우 안내

#### 제공하지 않는 내용

- 실시간 소요 시간 예측
- 교통 혼잡도 예측
- 택시/카셰어링 연계

#### 예시

```
📍 출발: 강남역
📍 도착: 여의도역

❌ 평소 이용: 간선버스 146번 (파업 영향)

✅ 대체 수단
1. 지하철 2호선 → 9호선 환승 (약 30분)
2. 광역버스 M버스 (파업 영향 없음)
3. 도보 10분 + 지하철 3호선 이용
```

## 9. 엣지 케이스 처리

### 9-1. 정보 변동 상황

| 상황                     | 처리 방법                                   |
| ------------------------ | ------------------------------------------- |
| **파업 예고 → 취소**     | 예고 알림 받은 사용자에게만 취소 알림 발송  |
| **파업 규모 확대**       | 업데이트 알림 발송 + 상세 페이지 갱신       |
| **파업 규모 축소**       | 업데이트 알림 발송 + 상세 페이지 갱신       |
| **출처 간 정보 상충**    | 1차 출처(시/도청) 우선, 상충 내용 명시 표시 |
| **연속 파업 (2일 이상)** | 첫날만 알림, 상세 페이지에 기간 표시        |

---

### 9-2. 잘못된 정보 발송 시 대응

**프로세스**

1. 긴급 정보 수정 알림 발송 (1시간 이내)
2. 상세 페이지 상단에 수정 내역 표시
3. 사후 분석 및 재발 방지책 수립

**알림 문구 예시**

```
[정보 수정] 이전 파업 알림이 수정되었습니다
→ 최신 정보 확인하기
```

---

### 9-3. 기술 장애 상황

| 상황                    | Fallback 방안                                     |
| ----------------------- | ------------------------------------------------- |
| **정보 수집 실패**      | 마지막 확인된 정보 + "정보 업데이트 지연 중" 표시 |
| **푸시 알림 발송 실패** | 재발송 시도 (최대 3회) + 인앱 알림으로 대체       |
| **서버 다운**           | 정적 페이지로 fallback (마지막 업데이트 정보)     |

## 10. 개인정보 및 법적 고려사항

### 10-1. 개인정보 수집 항목

| 항목               | 필수/선택 | 용도           |
| ------------------ | --------- | -------------- |
| **푸시 알림 권한** | 필수      | 파업 알림 발송 |
| **위치 정보**      | 선택      | 지역 맞춤 알림 |
| **출퇴근 루트**    | 선택      | 대체 수단 안내 |

**개인정보처리방침**

- 위치 정보: "구/시" 단위까지만 수집 (정확한 주소 미수집)
- 출퇴근 루트: 암호화 저장, 서비스 목적 외 사용 금지
- 사용자 요청 시 즉시 삭제 가능

---

### 10-2. 서비스 책임 범위

**면책 조항**

- 파업 정보는 공식 기관 발표를 기반으로 제공
- 정보 변동에 따른 피해 책임은 서비스 제공자가 지지 않음
- 대체 이동 수단은 참고 정보이며, 실제 소요 시간과 차이 있을 수 있음

**서비스 이용 약관 명시 필요**

```
본 서비스는 공식 기관의 파업 정보를 전달하는
안내 서비스이며, 정보의 정확성이나 실시간성을
보장하지 않습니다.
이동 계획 수립 시 공식 발표를 재확인해주시기 바랍니다.
```

## 11. 성공 지표 (KPI)

### 11-1. 핵심 가치 지표

| 지표                                     | 목표                    | 측정 방법                           |
| ---------------------------------------- | ----------------------- | ----------------------------------- |
| **파업 알림 수신 후 상세 페이지 조회율** | 70% 이상                | 푸시 클릭 수 / 푸시 발송 수         |
| **정보 유용성 피드백 긍정 비율**         | 80% 이상                | 상세 페이지 내 "도움됨" 버튼 클릭률 |
| **정보 제공 속도**                       | 공식 발표 후 1시간 이내 | 발표 시각 - 알림 발송 시각          |

**피드백 수집 방법**

- 상세 페이지 하단에 간단한 피드백 버튼 배치
  ```
  이 정보가 도움이 되셨나요?
  [👍 도움됨] [👎 아니요]
  ```
- 부정 피드백 시 간단한 사유 수집 (선택 항목)

| ------------------------------ | -------- | -------------------------- |

### 11-3. 가드레일 지표 (부정 지표)
