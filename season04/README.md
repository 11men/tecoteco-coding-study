## 🎯 시즌 4 소개

알고리즘 정답을 찾던 1~3시즌과는 달리, 시즌 4에서는 **풀어야 할 문제를 발견하고 AI와 함께 프로토타입을 만드는 실전 스프린트**를 진행합니다.

- **문제 발견**: 세상의 불편함을 함께 도출하고 해결할 문제 선정

- **프로토타이핑**: 2시간 안에 '눈으로 보고 클릭할 수 있는' 결과물 제작
- **팀 로테이션**: 매주 새로운 팀원과 협업하며 다양한 시너지 경험

## 💡 테코테코 스프린트 3원칙

1. **AI를 파트너로 삼아요**
   막힐 때만 쓰는 검색 도구가 아니에요. 문제 발견, 기획, 프로토타이핑의 모든 과정에서 AI를 동료처럼 활용하며, 더 나은 방법을 함께 찾아요.

2. **완성보다 과정을 즐겨요**
   완벽한 제품을 만드는 게 목표가 아니에요. '왜?'라고 질문하고, 동료와 토론하며, 함께 해결책을 찾아가는 경험 그 자체가 우리의 결과물이에요.

3. **서로를 존중하고 경청해요**
   매주 새로운 동료와 최상의 시너지를 내기 위해, 비판이 아닌 비평을, 주장보다 질문을, 정답 대신 다양한 의견을 존중해요.

## 📝 산출물 등록 가이드

### 1️⃣ 회차 폴더 및 README 생성

각 회차가 끝난 후, 해당 회차 폴더와 README를 작성합니다.

```bash
# 예시: 1회차 폴더 구조 생성
mkdir -p session-01
cd session-01
touch README.md
```

**회차 README 작성 시 포함할 내용:**
- 날짜 및 참가자
- 도출된 문제 목록
- 선정된 문제 및 팀 배정
- 전체 회고

👉 [SESSION_README_template.md](./templates/SESSION_README_template.md) 참고

### 2️⃣ 팀별 프로젝트 폴더 생성

각 팀은 다음과 같은 네이밍으로 프로젝트 폴더를 생성합니다:

```
team-{A/B/C}-{프로젝트-간단명}/
```

예시:
- `team-A-meeting-todo/` (회의록 TODO 추출기)
- `team-B-schedule-helper/` (스케줄 조율 도우미)

### 3️⃣ 프로젝트 README 작성

각 팀 프로젝트 폴더 내 README.md에 다음 내용을 작성합니다:

- 프로젝트 정보 (회차, 팀원, 해결 문제)
- 솔루션 설명
- 핵심 기능
- 스크린샷/데모
- 기술 스택
- 배운 점 / 회고
- 향후 개선 방향

👉 [PROJECT_README_template.md](./templates/PROJECT_README_template.md) 참고

### 4️⃣ 코드 및 자료 업로드

프로젝트 폴더 내에 소스코드, 스크린샷, 데모 영상 등을 업로드합니다.

```
team-A-meeting-todo/
├── README.md
├── src/
│   ├── index.html
│   ├── app.js
│   └── ...
├── screenshots/
│   ├── demo-01.png
│   └── demo-02.png
└── demo.mp4  (선택사항)
```
