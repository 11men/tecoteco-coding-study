# 밈 사전 (Meme Dictionary for Marketers) - 요구사항 정의서

---

## 1. 프로젝트 개요

### 1.1 프로젝트명
밈 사전 (Meme Dictionary for Marketers)

### 1.2 핵심 가치 제안
"트렌드를 놓치지 않고, 실전에서 바로 쓸 수 있는 밈 백과사전 + 이모티콘 생성 도구"

### 1.3 타겟 사용자
| 구분 | 대상 |
|------|------|
| 1차 타겟 | 마케터, 콘텐츠 크리에이터, 브랜드 매니저 |
| 2차 타겟 | 트렌드에 관심 있는 일반 사용자 |

### 1.4 기존 서비스 대비 차별화
| 기존 서비스 | 밈 사전 |
|-------------|---------|
| 신조어 사전 (단순 의미) | 마케팅 실전 가이드 |
| 정적인 정보 나열 | 실시간 트렌드 점수 |
| 텍스트만 제공 | 바로 쓸 수 있는 이모티콘 |
| 검색만 가능 | 밈고사로 재미 + 학습 |
| 일반 사용자 대상 | 마케터 특화 |

## 3. 핵심 기능 상세

### 3.1 밈 백과사전

#### 3.1.1 검색 및 분류 체계
- **플랫폼별**: 인스타그램 / 틱톡 / X(트위터) / 유튜브 / 커뮤니티(디시, 에펨코, 더쿠 등)
- **카테고리별**:
  - **유행어**: 단어/표현 (예: "~하다못해", "제곱근 하트"), 신조어 (예: "갓생", "워라밸")
  - **유행템**: 패션/라이프스타일 아이템 (예: 스탠리 텀블러)
  - **유행 음식**: 식품 트렌드 (예: 마라탕후루)
  - **유행 챌린지**: 소셜미디어 챌린지 (예: Random Play Dance)
  - **유행 음악**: 바이럴 음악 (예: APT. - ROSE & Bruno Mars)
- **시대별**: 2010년대 ~ 2025년 타임라인
- **마케팅 활용도별**: 브랜드 카피 적합 / 이벤트 적합 / SNS 콘텐츠 적합 / 사용 주의

#### 3.1.2 밈 상세 페이지 구성
각 밈 항목은 다음 정보를 포함한다:

| 섹션 | 내용 |
|------|------|
| 기본 정보 | 카테고리, 출처 플랫폼, 유행 시기, 트렌드 점수(100점 만점, 상승/하락 표시) |
| 의미 & 유래 | 밈 설명, 원본 영상 임베드, 스크린샷 갤러리, 확산 타임라인 시각화 |
| 마케팅 활용 가이드 | 적합 산업, 비추천 산업, AI 생성 카피 예시 3개, 시즌 연결(적합도 별점), 리스크 알림 |
| 실제 활용 사례 | 브랜드 활용 사례 및 성과 |
| 밈 이모티콘 | 이모티콘 갤러리, AI 생성 요청 버튼, 커뮤니티 제작 이모티콘 |
| 관련 밈 | 연관 밈 링크 |

#### 3.1.3 원조 출처 트래킹 시스템
- **원본 링크 임베드**: YouTube oEmbed, Instagram Embed API, TikTok oEmbed, X Embed
- **백업 스냅샷** (원본 삭제 대비): 영상 썸네일 저장, 핵심 장면 스크린샷 3~5장, 텍스트 설명 백업
- **확산 타임라인**: 최초 등장 -> 플랫폼 확산 -> 챌린지화 -> 브랜드 활용 -> 주류 미디어 보도

#### 3.1.4 밈 생명주기 그래프
- 트렌드 점수의 시계열 시각화 (월별)
- 현재 위치 표시 (상승기/정점/하락기)
- 예상 잔여 수명 표시

---

### 3.2 밈고사 (퀴즈) 시스템

#### 3.2.1 진행 플로우
1. 사용자 진입: 년도별 밈고사 선택
2. 10문제 출제 (객관식 4지선다)
3. 점수 계산 (100점 만점)
4. 등급 부여
5. 불합격 시 전 세대 고사 재응시 안내
6. 결과 공유: SNS 공유용 이미지 자동 생성

#### 3.2.2 등급 시스템
| 점수 | 등급 | 설명 |
|------|------|------|
| 90~100 | 트렌드 선구자 | 밈 고고학자 수준 |
| 70~89 | MZ 인싸 | 최신 트렌드 따라잡는 중 |
| 50~69 | 밈 뉴비 | 아직 공부가 필요해요 |
| 0~49 | 밈 문맹 | 전 세대 고사를 추천합니다 |

#### 3.2.3 시대별 고사 구성
- 2025년 밈고사 (최신)
- 2024년 밈고사
- 2020~2023 통합 고사 (구제 고사, 더 쉬운 난이도)
- 2010년대 레트로 고사 (밈 역사 탐험)

### 3.3 밈 이모티콘 스튜디오

#### 3.3.1 AI 이모티콘 생성
- **입력**: 텍스트 프롬프트 + 참고 이미지 업로드(선택)
- **처리**: AI 이미지 생성 엔진 (DALL-E 3 / Stable Diffusion)
- **출력**: 4개 옵션 제시 -> 사용자 선택 -> 다운로드 (PNG/SVG)

#### 3.3.2 커뮤니티 이모티콘
- 디자이너/일반 사용자가 직접 제작 후 업로드
- 관리자/AI 검증 (저작권, 퀄리티 체크)
- 승인 후 밈 상세 페이지에 등록
- 인기 이모티콘 TOP 10 큐레이션

#### 3.3.3 커스터마이징 도구
- 기본 이모티콘 선택 후 편집:
  - 색상 변경 (브랜드 CI 적용)
  - 텍스트 추가 ("SALE", "NEW" 등)
  - 표정/크기 조절
  - 배경 추가/제거
- 실시간 프리뷰 -> 다운로드

#### 3.3.4 밈 조합 생성기
- 두 개 밈 선택 -> AI가 조합 이모티콘 자동 생성
- 예: "제곱근 하트" + "워라밸" -> "sqrt(워라밸) 달성!" 이모티콘

#### 3.3.5 라이선스 시스템
각 이모티콘에 라이선스 유형 명시:
| 유형 | 설명 |
|------|------|
| CC0 | 상업적 사용 가능 |
| CC BY | 출처 표기 필수 |
| Personal Use | 개인용만 가능 |
| 유료 라이선스 | 저작권자 연락 필요 |

---

### 3.4 마케팅 도구

#### 3.4.1 AI 카피 제네레이터
- **입력**: 밈 선택 + 브랜드 정보 + 캠페인 목적
- **출력**: 3가지 카피 변형안
- 예시: 밈 "제곱근 하트" + "친환경 화장품" + "발렌타인데이"
  - "sqrt(자연)에서 온 사랑의 선물"
  - "sqrt(순수)한 마음을 담았습니다"
  - "고객님께 sqrt(감사)를 전합니다"

#### 3.4.2 시즌 캘린더
- 현재 시점 기반 자동 추천
- 예: 11월 -> 블랙프라이데이/크리스마스/연말 결산용 밈 추천

#### 3.4.3 리스크 체커
- 논란 밈 자동 필터링
- 논란 사유, 발생 일시, 안전한 대체 밈 3개 추천 제공

#### 3.4.4 산업별 필터
- 사용자가 "내 산업" 설정 (예: 패션 브랜드)
- 대시보드에 산업 맞춤 밈 자동 큐레이션
- 해당 산업 실제 활용 사례 및 타겟 맞춤 밈 제공

#### 3.4.5 경쟁 브랜드 벤치마킹
- 해당 밈을 활용한 브랜드 목록 및 성과 데이터 제공
- 예: "A브랜드: 인스타 릴스 조회수 100만", "B브랜드: 이벤트 참여율 300% 증가"

## 4. 기술 스택

### 4.1 Frontend
| 항목 | 기술 |
|------|------|
| Framework | Next.js 14 (App Router) |
| 스타일링 | TailwindCSS |
| 컴포넌트 | Radix UI / shadcn/ui |
| 이모티콘 편집 | Fabric.js (캔버스 편집), react-colorful (색상 선택) |
| 애니메이션 | Framer Motion |
| 상태 관리 | Zustand / React Query |

### 4.2 Backend
| 항목 | 기술 |
|------|------|
| Framework | FastAPI (Python) 또는 Node.js (Express/Nest.js) |
| Database | PostgreSQL (밈 데이터, 사용자) |
| 캐싱 | Redis (트렌드 점수) |
| Storage | AWS S3 / Cloudflare R2 (이모티콘 파일) |
| CDN | Cloudflare |

### 4.3 AI & Data
| 항목 | 기술 |
|------|------|
| LLM | OpenAI GPT-4 (카피 생성, 밈 설명), Anthropic Claude (긴 컨텍스트 분석) |
| 이미지 생성 | DALL-E 3 (이모티콘 생성), Stable Diffusion (오픈소스 대안) |
| 크롤링 | Puppeteer/Playwright (웹), Instagram API / TikTok API (공식 API) |
| 임베드 | YouTube oEmbed API, Instagram Embed API, TikTok oEmbed API |

### 4.4 인프라
| 항목 | 기술 |
|------|------|
| Hosting | Vercel (Frontend) + AWS/Railway (Backend) |
| CI/CD | GitHub Actions |
| Monitoring | Sentry (에러), Posthog (분석) |
